How BongoScript Works

BongoScript programs go through a multi-stage compilation pipeline. The source is tokenized, parsed, transpiled into C, compiled with GCC, and executed — all in the browser.

Compilation Pipeline

Your BongoScript source code is first tokenized by a Flex-generated lexer, then parsed by a Bison-generated parser that emits equivalent C code. The generated C file is compiled with GCC and executed on the server.

Lexer — Flex

Tokenizes BongoScript source into keywords like dhoro, dekhao, nao, identifiers, numbers, and operators.

Parser — Bison

Defines the grammar rules for BongoScript and generates equivalent C code through semantic actions in the parse tree.

Backend — Node.js

An Express server receives code from the browser, runs the transpiler and GCC, and returns the program output back to the client.

Execution — GCC

The transpiled C file (output.c) is compiled into a native binary and executed. Standard output is captured and returned.

Language Reference

A quick reference for all BongoScript keywords and operators. All keywords are Bangla transliterations of common programming constructs.

Keyword Description Example
dhoro Declare an integer variable dhoro x = 10
dekhao Print a value or string to stdout dekhao x + y
dekhao "..." Print a string literal dekhao "hello world"
nao Read input from the user nao x
jodi / nahole If / Else conditional block jodi x > 5:
loop While loop block loop i < 10:
+ - * / == > < Operators (arithmetic & comparison) dekhao a * b + c
dhoro x[5] Declare an integer array dhoro arr[10]
